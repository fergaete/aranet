link_to_styled (line 29)

Automatically adds a class name to the link if the link route matches the current module or route.

Style Options:

  • 'class' - the class name to append to the link, defaults to 'current'
  • 'module_only' - append class name if the current module and link module match
  • 'tag' - an html to encapsulate the link, which will receive the class name rather than the <a> tag
Examples:
  1.    echo link_to_styled('Events', 'event/list', 'class=active', 'id=event_link', );
  2.    if the current route is 'event/list', then:
  3.      => <a href="path/to/event/list/action" id="event_link" class="active">Events</a>
  4.  
  5.    echo link_to_styled('Events', 'event/list', 'class=active module_only=true tag=li', 'id=event_link');
  6.    if the current module matches the module present in the route, then:
  7.      => <li class="active"><a href="path/to/event/list/action" id="event_link">Events</a></li>

  • return: XHTML compliant <a href> tag
  • see: link_to
string link_to_styled (string $text, [string $route = ''], [array $style_options = ''], [array $options = ''])
  • string $text: name of the link, i.e. string to appear between the <a> tags
  • string $route: 'module/action' or '@rule' of the action
  • array $style_options: additional style options
